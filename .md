State (useState) Updates automatically, causes re-render

Re-render = React updates the UI with new values

Ref (useRef) Store value, doesn't update automatically, doesn't cause re-render

State (useState) → Update causes re-render (UI updates)
Ref (useRef) → Update does NOT cause re-render (UI doesn't update)

That's why useRef is used for tracking old values without affecting the UI! 👍

pointer-events-auto = Allow clicks/interactions on this element

pointer-events-none = Block clicks/interactions (ignore all clicks)

opacity-100 = Fully visible (100% opacity)

opacity-0 = Invisible (0% opacity)

fixed = Position fixed to viewport (doesn't scroll) 
inset-0 = Cover entire screen (top:0, bottom:0, left:0, right:0)

ref = gives you direct access to a DOM element in JavaScript
javascriptconst mobileMenuRef = useRef(null)
Creates a reference to store the element.

`<div ref={mobileMenuRef}>Mobile Menu</div>`
Now you can access and control it:
// Get the element
mobileMenuRef.current

// Change its style
mobileMenuRef.current.style.display = 'none'

// Add/remove classes
mobileMenuRef.current.classList.add('hidden')

// Get its properties
mobileMenuRef.current.offsetWidth

In context (mobile menu):

Menu slides in from right when open
Menu slides out to right when closed
Clickable inside menu without closing it
Can be controlled via ref


transform: translateX(${isOpen ? 0 : '20px'})`

When menu opens → translateX(0) (items at normal position)
When menu closes → translateX(20px) (items moved 20px to the right)
Items slide in from right when opening
